# Nest.js CLI를 활용한 모노레포 구성하기

대규모 애플리케이션 개발에 있어서 코드의 재사용성과 관리는 매우 중요합니다. <br />
이러한 문제를 해결하는 하나의 방법으로 모노레포(Monorepo) 구조를 사용할 수 있습니다.

모노레포는 여러 프로젝트의 코드를 한 개의 저장소에서 관리할 수 있게 해주어, 효율적인 코드 관리와 협업을 가능하게 합니다 이 글에서는 Nest.js CLI를 활용하여 모노레포 구조를 설정하는 방법에 대해 설명합니다.

## Nest.js CLI 설치

Nest.js CLI는 프로젝트를 생성하고, 모노레포 구조를 손쉽게 구성할 수 있는 명령어를 제공합니다.

모노레포 구성을 시작하기 전에, 시스템에 Nest.js CLI가 설치되어 있어야 합니다. CLI 설치는 다음과 같은 명령어로 수행할 수 있습니다

```bash title="Nest.js CLI 설치"
npm install -g @nestjs/cli

# 또는 yarn을 사용하는 경우
yarn global add @nestjs/cli
```

## 모노레포 구성 시작하기

1. **새 워크스페이스 생성**: 먼저, 새 Nest.js 워크스페이스를 생성합니다. 워크스페이스는 모노레포를 구성하는 데 사용될 최상위 폴더입니다.

```bash
nest new workspace-name --monorepo
```

여기서 `workspace-name`은 워크스페이스의 이름입니다.

2. **애플리케이션 추가**: 워크스페이스 내부에서 다수의 Nest.js 애플리케이션을 생성할 수 있습니다. 애플리케이션을 추가하려면 다음 명령어를 사용합니다

```bash
nest generate app app-name
```

`app-name`은 추가하려는 애플리케이션의 이름입니다.

3. **라이브러리 추가**: 워크스페이스 내부 여러 앱들의 공통 기능을 재사용하기 위한 라이브러리를 추가할 수도 있습니다. 라이브러리를 추가하려면 다음 명령어를 실행합니다

```bash
nest generate library lib-name
```

`lib-name`은 생성하려는 라이브러리의 이름입니다.

## 프로젝트 구조 및 빌드

모노레포 구조에서는 각 애플리케이션과 라이브러리가 독립적으로 빌드 및 실행 될 수 있습니다.

프로젝트의 루트에서 다음 명령어를 사용하여 특정 애플리케이션을 빌드할 수 있습니다

```bash
nest build app-name

nest start app-name
```

또한, 모든 애플리케이션과 라이브러리를 한 번에 빌드하는 것도 가능합니다

```bash
nest build
```

## 결론

Nest.js CLI를 사용하여 모노레포 구조를 설정하는 것은 대규모 프로젝트 관리를 간소화하고, 코드 재사용성을 높이는 데 큰 도움이 됩니다. 이 글을 통해 Nest.js를 사용한 모노레포 구성의 기본적인 방법을 이해하고, 자신만의 프로젝트에 적용해 볼 수 있기를 바랍니다.
